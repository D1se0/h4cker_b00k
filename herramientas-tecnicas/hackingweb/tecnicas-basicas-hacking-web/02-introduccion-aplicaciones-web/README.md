# Introducción a las Aplicaciones Web

Antes de atacar cualquier aplicación, hay que entender cómo está construida. Este bloque sienta las bases conceptuales: qué diferencia una aplicación web de un sitio estático, cómo se reparten las responsabilidades entre frontend y backend, qué arquitecturas e infraestructuras existen detrás de una web moderna, y un primer recorrido por las vulnerabilidades más habituales del ecosistema web — la antesala de todo lo que se trata en profundidad en el resto de los bloques.

## Por qué este bloque importa

Muchos pentesters principiantes saltan directamente a "probar payloads" sin entender qué hay detrás. Pero saber identificar la pila tecnológica de una aplicación (¿es PHP+MySQL? ¿Node+MongoDB? ¿usa una API REST?) condiciona completamente qué vulnerabilidades son plausibles y cómo se explotan. Este bloque es el mapa mental que se usa constantemente en el resto del temario.
