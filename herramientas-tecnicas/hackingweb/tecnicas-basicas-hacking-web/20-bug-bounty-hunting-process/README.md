# Bug Bounty Hunting Process

El último bloque de este temario cierra el círculo: una vez que se ha aprendido a identificar y explotar vulnerabilidades, ¿cómo se reportan de forma que sean tomadas en serio, recompensadas, y corregidas? Este bloque cubre los fundamentos de los programas de bug bounty, cómo funciona el proceso de participación, y cómo redactar informes de vulnerabilidades que comuniquen el hallazgo de forma clara, reproducible, y profesional.

## Por qué este bloque cierra el temario

Saber encontrar vulnerabilidades es solo la mitad del trabajo. Un hallazgo que no se comunica de forma clara, reproducible, y con el impacto bien articulado tiene pocas probabilidades de ser corregido a tiempo — o de ser reconocido y recompensado en un programa de bug bounty. Este bloque es el puente entre "sé explotar esto" y "puedo comunicarlo de forma que marque una diferencia real".
