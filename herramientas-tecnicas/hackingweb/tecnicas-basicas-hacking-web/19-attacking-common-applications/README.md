# Attacking Common Applications

Las aplicaciones empresariales ampliamente desplegadas — CMS, servidores de aplicaciones, herramientas de CI/CD, sistemas de monitorización — son objetivos de alta prioridad en auditorías de seguridad precisamente porque su popularidad implica que sus vulnerabilidades son ampliamente conocidas y documentadas. Este bloque cubre la metodología de reconocimiento y auditoría para las plataformas más frecuentes en entornos corporativos.

## Por qué este bloque importa

Una vulnerabilidad en WordPress, Tomcat, o Jenkins puede ser el punto de entrada a una red corporativa completa — especialmente cuando estos sistemas se despliegan con configuraciones por defecto, credenciales no cambiadas, o plugins/extensiones desactualizados. El reconocimiento efectivo de qué aplicaciones comunes están en uso es uno de los primeros pasos de cualquier auditoría externa.
